# Storage verify/repair (CLI)

## Команды
- Проверка доступности файлов (диск/БД):  
  `cd backend && npm run storage:verify [-- --limit=100]`
- Проверка с попыткой восстановления на диск из БД/дубликатов:  
  `cd backend && npm run storage:repair [-- --limit=100]`

## Что делает
- Обходит statements (опционально с `--limit`).
- Проверяет наличие файла на диске и в `fileData`.
- При `--repair` восстанавливает копию на диск, если есть данные в БД или другой statement с тем же hash.
- Итоговый отчёт: всего, healthy, missing, repaired; список первых missing.

## Выходные коды
- `0` — проверка выполнена, отчёт в stdout.
- `!=0` — критическая ошибка запуска.
