# Настройки профиля (что ещё добавить)

Документ описывает, какие блоки стоит добавить в **Настройки профиля** помимо текущих полей **Email** и **Пароль**.

## 1) Личные данные
- **Имя/ФИО** (отображаемое имя в интерфейсе и в приглашениях).
- **Аватар** (опционально).
- **Телефон** (опционально; если используется Telegram/SMS).
- **Часовой пояс** (важно для дат в отчётах/уведомлениях).
- **Язык интерфейса** (ru/en/kk) + формат даты/чисел.

## 2) Безопасность аккаунта
- **Сеансы и устройства**
  - список активных сессий (устройство, браузер, IP, время),
  - кнопка “Выйти со всех устройств”.
- **2FA / MFA**
  - TOTP (Google Authenticator и аналоги),
  - резервные коды,
  - принудительное включение 2FA для админов (опционально).
- **История входов**
  - успешные/неуспешные входы, подозрительная активность.
- **Политика паролей**
  - требования, подсказка по сложности,
  - защита от частых попыток (rate limit уже есть, но стоит показать UX).

## 3) Уведомления
- Каналы: **Email**, **Telegram**, (в будущем) Push.
- Гранулярные переключатели:
  - “Выписка загружена/обработана”,
  - “Ошибки парсинга”,
  - “Еженедельный/ежемесячный отчёт”,
  - “Приглашения в рабочее пространство”,
  - “Поделиться файлом / доступ выдан”.
- Тихие часы (do-not-disturb) и частота дайджестов.

## 4) Интеграции
- **Telegram** (у вас уже есть отдельная страница, но в профиле удобно иметь “Состояние”):
  - подключено/не подключено, ID/username (без лишней чувствительной инфы),
  - кнопка переподключить/отвязать.
- **Google Sheets**
  - подключение аккаунта, список привязанных таблиц,
  - управление доступом/правами интеграции,
  - отключить интеграцию.
- (Опционально) Webhooks / API токены для интеграций (если планируется).

## 5) Настройки данных и приватности
- **Экспорт данных**
  - выгрузка выписок/операций/категорий/таблиц в CSV/XLSX/JSON,
  - статус экспорта, ссылка на скачивание.
- **Импорт данных**
  - источники, история импортов, ошибки.
- **Удаление аккаунта**
  - “Удалить аккаунт” с подтверждением,
  - выбор: удалить всё / оставить рабочее пространство (если есть владелец) / передать владельца.
- **Согласия**
  - маркетинговые письма (если будут),
  - аналитика/телеметрия (если используется).

## 6) Настройки приложения (UX)
- Тема: светлая/тёмная/системная.
- Настройки таблиц по умолчанию:
  - валюта по умолчанию,
  - формат чисел (разделители),
  - дефолтные фильтры/период в отчётах.
- Горячие клавиши (если появятся) / подсказки.

## 7) Командный контекст (ссылка на workspace)
В профиле удобно показывать:
- текущее рабочее пространство (название),
- роль пользователя,
- быстрый переход в “Рабочее пространство”.

## 8) Админские/служебные секции (только для админов/владельцев)
- Аудит-лог действий (кто/что/когда) — хотя это скорее “Настройки рабочего пространства”.
- Управление доступом: роли/права (у вас уже расширяется в workspace).

## MVP-рекомендация (минимум, который даст максимальный эффект)
1) Имя + язык + часовой пояс  
2) Сеансы (выйти со всех устройств)  
3) Уведомления (Email/Telegram)  
4) Экспорт данных + удаление аккаунта
